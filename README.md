# Banka - Banking Web Application

A Flask-based banking web application connected to Oracle Database that allows users to manage financial accounts, perform transactions, and transfer money between accounts.

## Features

### User Features
- **Secure Login**: Authenticate using P-tal (personal ID) and password
- **Account Dashboard**: View all personal accounts and family account balance
- **Transactions**: 
  - Deposit money (Set Inn)
  - Withdraw money (Tak út)
  - View all transaction history (Bokingar)
- **Money Transfers**: 
  - Transfer money between accounts (Flyt Pengar)
  - View all transfer history (Flytingar) with recipient information

### Admin Features
- **User Management**: 
  - Create new customers (Legg afturat persón)
  - View all customers with full details (Kundar)
- **Account Management**:
  - Create new accounts for existing customers (Legg afturat konto)
  - View all accounts across the system
- **Full System Visibility**:
  - View all transactions and transfers in the system
  - Access to all customer and account data

## Technology Stack

- **Backend**: Flask (Python web framework)
- **Database**: Oracle Database 19c
- **Database Driver**: oracledb
- **Language**: Faroese (UI)
- **Runtime**: Python 3.x

## Setup Instructions

### Prerequisites
- Python 3.7 or higher
- Oracle Database 19c or compatible
- Oracle SQL*Net connectivity to `192.168.9.176/xepdb1`

### Installation

1. Clone or download the project to your machine
2. Navigate to the project directory:
   ```powershell
   cd c:\Users\teodo\Desktop\Banka
   ```

3. Create a virtual environment:
   ```powershell
   python -m venv .venv
   ```

4. Activate the virtual environment:
   ```powershell
   .\.venv\Scripts\Activate
   ```

5. Install required packages:
   ```powershell
   pip install flask oracledb
   ```

6. Run the application:
   ```powershell
   python app.py
   ```

7. Open your browser and navigate to:
   ```
   http://localhost:5000
   ```

## Database Schema

### Core Tables

| Table | Description |
|-------|-------------|
| **personur** | Stores personal information (name, birthdate, address, postal code, gender) |
| **kundi** | Links persons to the banking system with kundi_id |
| **konto** | Bank accounts with account number, type, and balance |
| **boking** | Transaction records (deposits/withdrawals) |
| **kladda** | Money transfer records between accounts |
| **familja** | Family group definitions |
| **familju_limir** | Family group members |
| **konto_slag** | Account type classification (Debit/Kredit/Trota) |
| **postkoda** | Postal codes and city names |

### Key Columns

**personur**:
- `ptal` - Personal ID (auto-generated by `ptal_gen()` function)
- `fornavn` - First name
- `eftirnavn` - Last name
- `fodidato` - Birth date
- `adressa` - Address
- `postkoda` - Postal code
- `kyn` - Gender (Maður/Kvinna)

**konto**:
- `kontonr` - Account number (auto-generated)
- `kundi_id` - Link to customer
- `konto_slag` - Account type
- `saldo` - Current balance

**boking**:
- `bokingar_id` - Transaction ID
- `kontonr` - Account number
- `dato` - Transaction date
- `upphaedd` - Amount
- `bokingar_slag` - Transaction type (Deposit/Withdrawal)

**kladda**:
- `kladda_id` - Transfer ID
- `kontonr_fra` - Sending account
- `kontonr_til` - Receiving account
- `dato` - Transfer date
- `upphaedd` - Transfer amount
- `saldo_fra` - Sender balance at time of transfer
- `saldo_til` - Receiver balance at time of transfer

## Key Features Explained

### Family Accounts
Regular users see both their own accounts and family member accounts when logged in. This is determined by the `familja` and `familju_limir` tables.

### Transaction History
- **Bokingar (Transactions)**: Shows all deposits (Set Inn) and withdrawals (Tak út)
- **Kladda (Transfers)**: Shows money transfers between accounts with recipient information

### Admin Dashboard
Admins have additional buttons on the dashboard:
- Create new persons/customers
- Add new accounts
- View all customers in the system

## Important Notes

- All dates are displayed in YYYY-MM-DD format
- User passwords are currently universal (hardcoded as "password") for testing purposes
- P-tal values are unique and auto-generated using the database function `ptal_gen()`
- Account numbers are auto-generated using the database function `konto_gen()`
- The application runs in debug mode for development

## Troubleshooting

### Database Connection Error
- Verify Oracle Database is running and accessible at 192.168.9.176:1521
- Check that the SQL*Net configuration includes the xepdb1 connection alias
- Confirm database credentials are correct

### P-tal Not Recognized
- Ensure the P-tal exists in the `personur` table
- The password must be exactly "password"

### Account Not Found
- Verify the account exists in the `konto` table
- Check that sufficient balance exists for withdrawals
